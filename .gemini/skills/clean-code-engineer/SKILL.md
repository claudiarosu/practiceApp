# Clean Code Engineer

## Overview
This skill ensures that all code produced is of the highest quality, adhering to industry-standard best practices and local project conventions. It focuses on maintainability, readability, and performance.

## Core Guidelines

### 1. General Clean Code
Prioritize clarity and simplicity.
- **Reference:** See [general-coding.md](references/general-coding.md) for naming, functions, and error handling.

### 2. Node.js & TypeScript
Write idiomatic, type-safe backend and frontend code.
- **Reference:** See [nodejs-typescript.md](references/nodejs-typescript.md) for async patterns and Next.js App Router standards.

### 3. Performance & Optimization
Ensure the application remains fast and scalable.
- **Reference:** See [performance.md](references/performance.md) for Prisma optimization and React best practices.

## Workflow

1. **Planning:** Review the requirements and identify the most idiomatic way to implement them in the current stack.
2. **Implementation:** Write surgical, well-tested code. Use descriptive names and follow SRP (Single Responsibility Principle).
3. **Refactoring:** Before finishing, review the changes against the "Boy Scout Rule" to ensure the workspace is cleaner than before.
4. **Verification:** Always run type checks and tests to confirm integrity.

## Examples

- "Implement a new API endpoint for submitting parent reviews using Clean Code standards."
- "Refactor the school filtering logic to be more DRY and performant."
- "Review this component for any Node.js or TypeScript anti-patterns."
