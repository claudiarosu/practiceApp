# Software Design Architect

## Overview
This skill transforms Gemini CLI into a senior software architect. It provides structured guidance for building modern web applications, with a specialized focus on the Bucharest educational ecosystem (schools, kindergartens, and high schools).

## Capabilities

### 1. Domain Modeling
Helps define the core entities and relationships for your application.
- **Bucharest Context:** See [bucharest-context.md](references/bucharest-context.md) for local school system details.
- **Data Patterns:** See [data-models.md](references/data-models.md) for standard school and review schemas.

### 2. Architecture Planning
Guides the selection of tech stacks and system components.
- **Guidelines:** See [architecture-guidelines.md](references/architecture-guidelines.md) for recommendations on React, Next.js, and Geo-spatial databases.

### 3. Feature Design
Assists in planning complex features like school comparisons, proximity searches, and parent reviews.

## Workflow

1. **Research Phase:** Use the architect to analyze requirements (like those in your README.md).
2. **Strategy Phase:** Request a tech stack recommendation and high-level architecture.
3. **Design Phase:** Ask for detailed database schemas and API endpoint definitions.
4. **Implementation Phase:** Use the architect to scaffold boilerplate or review code against best practices.

## Examples

- "Help me design a database schema for a school search app in Bucharest."
- "What tech stack should I use for a fast, SEO-friendly parent portal?"
- "Explain the difference between Sector 1 and Sector 6 school demographics for our data model."
